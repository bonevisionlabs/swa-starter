Security Headers

HSTS, CSP, X-Frame-Options, Referrer-Policy, and Permissions-Policy pre-configured. Blocks sensitive file types by default.

SSO Authentication

Sign in with Microsoft, GitHub, or Google via Azure SWA's built-in auth. Role-based access control for protected routes.

Clean URLs, SPA fallback, auth redirects, and file blocking rules ready to customize for your app.

Optional Azure Functions backend in the api/ directory. Pre-configured in the workflow and route rules.

// Security
Hardened by default
Every response includes production-grade security headers. Configured in one JSON file.
staticwebapp.config.json 
{
  "globalHeaders": {
    "Strict-Transport-Security": "max-age=31536000",
    "X-Content-Type-Options": "nosniff",
    "X-Frame-Options": "SAMEORIGIN",
    "Content-Security-Policy": "default-src 'self'",
    "Referrer-Policy": "strict-origin...",
    "Permissions-Policy": "camera=(), ..."
  }
}
  • Force HTTPSHSTS with 1-year max-age and includeSubDomains
  • Prevent XSSContent Security Policy restricts script and style sources
  • Block clickjackingX-Frame-Options prevents embedding in iframes
  • Deny device APIsPermissions-Policy blocks camera, mic, and geolocation
  • Block sensitive filesRoutes return 404 for .py, .env, and .pem files
